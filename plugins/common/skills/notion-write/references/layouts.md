# 문서 유형별 레이아웃 레시피

문서 성격에 맞는 **골격**을 고른다. 공통 원칙: 상단에 식별(아이콘)·맥락(요약 콜아웃/메타)·내비(목차)를 세팅한 뒤 첫 헤딩으로 들어간다.

---

## 회의록

```
아이콘: 🗓️
<callout icon="🗓️" color="gray_bg">
	**일시** 2026-03-04 16:00 · **참석** 홍길동·성춘향 · **상태** 작성 중
</callout>

## 회의 목적
> 한 문장으로 이 회의가 왜 열렸는지.

## 이전 회의 리캡
- 지난 결정 요약(있으면)

## 논의
- 안건별 불릿

## 결정사항
- 확정된 것

## 다음 단계
- [ ] 액션 (담당: 이름)
- [ ] 액션 (담당: 이름)
```
- 목적=인용/콜아웃, 논의=불릿, 다음 단계=**체크박스+담당자**, 미해결 질문=볼드나 색 강조.
- 메타(일시·참석·상태)는 DB 페이지면 **속성**으로, 일반 페이지면 상단 콜아웃/표로.

---

## 가이드 / 매뉴얼 (산문형)

```
아이콘: ✔️
<callout icon="📋" color="gray_bg">
	**이 문서** 무엇을 다루는지 한 줄. **왜** 필요한지 한 줄.
</callout>
<table_of_contents/>

## 0. 개요
짧은 문단.

---

## 1. 첫 단계
설명 문단 + 필요 시 체크리스트.

---

## 2. 다음 단계
...

<details>
<summary>레거시(구버전) 방식 — 접어둠</summary>
	예전 절차. 지금은 안 쓰지만 참고용.
</details>
```
- 단계마다 `---`로 구획. 레거시·긴 곁가지는 토글로 접어 본문 흐름을 지킨다.
- 실행 항목은 체크리스트, 핵심 원칙은 인용/콜아웃.

## 보고서 / 분석 (역피라미드)

```
아이콘: 📊

	**결론** 가장 중요한 판단·권고를 먼저. **근거** 한 줄 요약.
</callout>
<table_of_contents/>

## 배경
왜 이 분석을 했는지.

## 분석
- 핵심 발견(볼드 리드인 + 설명)
- 데이터는 표로

## 결론 · 액션
- [ ] 권고 액션
```
- 결론을 맨 앞 콜아웃에 놓는다(스캔 사용자는 앞만 읽는다). 비교 데이터는 네이티브 표.

## Before / After 예시

같은 내용, 구조만 다르게.

### ❌ Before (텍스트 벽 + 코드펜스 감싸기)
페이지 전체가 하나의 코드펜스거나, 소제목·강조·구획 없이 이어진다:
```
파일 업로드 재시도 정책. 업로드가 실패하면 재시도를 해야 하는데 무한 재시도는
서버에 부담이 되므로 지수 백오프를 적용한다. 최대 3회까지 시도하고 그래도
실패하면 로컬 큐에 넣는다. 네트워크 오류와 5xx는 재시도 대상이고 4xx는
재시도하지 않는다. 로컬 큐는 다음 앱 실행 시 다시 시도한다. ...(계속)
```
문제: 요지가 어디 있는지 안 보이고, 재시도 대상/제외가 문장에 묻힌다.

### ✅ After (진입 앵커 + 청킹 + 표)
```
<callout icon="📋" color="gray_bg">
	**정책** 파일 업로드 실패 시 지수 백오프 3회 → 실패하면 로컬 큐 적재 후 다음 실행에 재시도.
</callout>
```
