## 요구사항

- Python 설치
- `gh` CLI 인증 완료
- Git 저장소 내에서 실행

## 규칙

- 코멘트 하나 처리 후 반드시 멈추고 사용자 승인 대기
- 여러 코멘트를 한꺼번에 수정 금지
- 코드 수정 전 반드시 해당 파일 Read
- 처리 완료한 코멘트는 컨텍스트 내 목록에서 완료 표시로 관리

## 작업 순서

### 1. PR 코멘트 수집

- `python "${SKILL_DIR}/scripts/main.py"` 실행 (`SKILL_DIR`은 이 스킬 파일이 위치한 디렉토리)
- PR 없으면 사용자에게 안내 후 종료
- 코멘트 없으면 사용자에게 안내 후 종료

### 2. 코멘트 목록 출력 및 선택 요청 [STOP]

아래 형식으로 출력 후 사용자에게 선택 요청:

```
## PR #N 리뷰 코멘트 목록

| # | 파일 | 문제 요약 | 난이도 |
|---|------|----------|--------|
| #N | `파일명:라인` | 한 줄 요약 | 🟢 쉬움 / 🟡 보통 / 🔴 복잡 |
...

💡 추천: #N - (추천 이유, 예: 버그 위험도가 높음 / 수정이 간단함)

어떤 코멘트부터 수정할까요? (번호 입력 또는 "추천대로")
```

- 우선순위 기준: 버그·race condition > 접근성 > 문서·주석
- "추천대로" 또는 Enter 입력 시 추천 코멘트로 진행

### 3. 코멘트 요약 및 승인 요청 [STOP]

아래 형식으로 요약 후 수정 여부 질문:

```
## Review Comment #N

**파일**: `경로/파일명:라인번호`
**문제**: (한 줄 요약)

### 문제 분석
- 현재 코드의 문제점 (코드까지 보여주면서 설명)
- 발생 가능한 영향 (예: 버그 발생, 성능 저하, 보안 취약점 등)

### 수정 방안
(구체적인 변경 내용 및 코드 예시)

수정할까요?
```

- 승인 → 4단계 진행
- 거절·건너뜀 → 2단계로 돌아가 다음 코멘트 선택

### 4. 코드 수정

- 해당 파일 Read 후 Edit으로 수정
- 수정 범위는 지적된 부분에 한정 (불필요한 리팩토링 추가 금지)
- 더 나은 방식이 있으면 이유 설명 후 사용자 확인
- 수정 완료 후 아래 명령으로 해당 스레드 resolve:
  ```bash
  gh api graphql -f query='
    mutation($threadId: ID!) {
      resolveReviewThread(input: {threadId: $threadId}) {
        thread { isResolved }
      }
    }
  ' -F threadId="<thread_id>"
  ```
  (thread_id는 스크립트 출력에 포함된 값 사용)

### 5. 커밋

- 커밋이 필요하면 사용자에게 `commit` 스킬 사용을 안내

### 6. 반복

- 모든 코멘트 처리 또는 사용자 중단 요청까지 2~5단계 반복
- 종료 시 처리 완료 수 / 남은 수 요약 출력
