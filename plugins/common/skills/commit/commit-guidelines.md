# 커밋 메시지 가이드라인

## 형식

```text
<type>: <변경 내용 한 줄 요약>

<선택: 구현 방식·설계 결정 이유 또는 문제 원인과 수정 방법. 개조식으로 작성>
```

## Type 목록

| type       | 사용 상황                  |
|------------|--------------------------|
| `feat`     | 새 기능 추가               |
| `fix`      | 버그·race condition 수정   |
| `refactor` | 기능 변경 없는 구조 개선    |
| `style`    | 접근성·UI 변경             |
| `test`     | 테스트 추가·수정            |
| `docs`     | 문서 작성·수정             |
| `chore`    | 빌드·설정 변경             |
| `ai`       | AI 지침·가이드라인·프롬프트 문서 변경 |
