# common

언어/플랫폼 무관 공통 개발 워크플로 도구.

## 스킬

| 스킬 | 설명 | 트리거 예시 |
| ---- | ---- | ----------- |
| `commit` | 변경사항을 분석해 커밋 메시지를 작성하고 git 커밋 수행 | "커밋해줘", "변경사항 정리해줘", "커밋 메시지 만들어줘" |
| `create-pr` | 현재 브랜치 변경사항으로 GitHub PR 생성·본문 업데이트 | "PR 만들어줘", "풀리퀘 올려줘", "PR 본문 업데이트해줘" |
| `apply-review` | PR 리뷰 코멘트를 가져와 보고서 작성 후 승인받아 코드 수정 | "PR 리뷰 반영해줘", "리뷰 코멘트 처리해줘", "리뷰 적용해" |
| `codebase-sync` | 코드베이스 현황 파악, 문서·코드 불일치 수정 후 커밋·PR까지 처리 | "현재 상황 파악해줘", "README 최신화해줘", "문서 동기화해줘" |
| `create-ppt` | 개발 진행 현황 브리핑용 HTML 프레젠테이션 생성 | "PPT 만들어줘", "발표자료 만들어", "브리핑 자료 생성해" |
| `draft-plan` | GitHub 이슈/주제를 받아 논의 후 구현 계획서를 `private/`에 작성 | "구현 계획서 작성해줘", "플랜 작성해줘", "이슈 #N 계획서" |
| `finish` | 정리 → 리뷰 → 커밋 → push → PR 개발 종료 체인을 한 번에 오케스트레이션 | "마무리해줘", "정리하고 커밋", "커밋하고 PR" |
| `memory-curator` | 세션 메모리 추출·저장(기본) / 누적 메모리 정리·승격 제안(리뷰) | "메모리 정리해줘", "세션 메모리 저장해줘", "메모리 리뷰" |
| `report` | 작업 내용을 프로젝트 `private/`에 마크다운 보고서로 작성 | "보고서 작성해줘", "작업 보고해줘", "완료 보고해줘" |
| `report-issue` | 자연어 설명으로 GitHub 이슈 초안 작성·생성 | "이슈 올려줘", "이슈 만들어줘", "버그 리포트" |
| `retrospective` | 대화 세션(또는 기간)을 회고해 인사이트를 스킬·메모리·볼트에 반영 | "회고해줘", "이번 작업 회고", "지난 주 회고" |
| `review-pack` | IDE 없이 문서 하나로 리뷰를 끝내는 "리뷰 팩" 생성 | "리뷰 팩 만들어줘", "리뷰하기 쉽게 정리해줘", "마지막으로 검토만 하자" |
| `setup-skills` | 프로젝트 기술 원칙 기반으로 맞춤 스킬 추천·생성 | "스킬 세팅해줘", "프로젝트에 맞는 스킬 만들어줘", "스킬 추천해줘" |
| `skill-metrics` | 세션 로그를 집계해 스킬 사용 빈도·자동화 우선순위 산출 | "스킬 사용 지표", "자동화 우선순위", "스킬 도달률" |

## 훅

| 훅 | 동작 |
| -- | ---- |
| `block_sensitive_files` | `.env`, 시크릿 키 등 민감한 파일 수정 차단 |
| `validate_commit` | 커밋 메시지 컨벤션 검증 |
